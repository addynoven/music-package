> **Pre-reqs**: Node ≥ 22, `yt-dlp` on PATH for stream/download fallback. Optional: `songrec` binary (Shazam), AcoustID API key, YouTube cookies for age-gated/private tracks.

## 1. Construction

### Recommended: async factory

```ts
import { MusicKit } from 'musicstream-sdk'
const mk = await MusicKit.create({ /* MusicKitConfig */ })
```

Use `MusicKit.create()` over `new MusicKit()` — it eagerly initialises the InnerTube client pool and lyrics registry so the first call is fast.

### Direct constructor (sync)

```ts
const mk = new MusicKit(config)
```

The lyrics registry is built lazily on first method call. Calling `mk.registerLyricsProvider()` before any other method has no effect — call after at least one method runs, or use `MusicKit.create()`.

### Custom audio source registration

```ts
mk.registerSource(myCustomAudioSource)  // implements AudioSource interface
```

### Custom lyrics provider registration

```ts
mk.registerLyricsProvider(provider)                  // append (default 'last')
mk.registerLyricsProvider(provider, 'first')         // prepend
mk.registerLyricsProvider(provider, 'before:lrclib') // ordered before
mk.registerLyricsProvider(provider, 'after:simpmusic') // ordered after
```

`RegistryPosition = 'first' | 'last' | 'before:<name>' | 'after:<name>'`

## 2. Configuration (`MusicKitConfig`)

Every key is optional. Pass to `MusicKit.create(config)` or `new MusicKit(config)`.

| Key | Type | Default | Purpose |
|---|---|---|---|
| `logLevel` | `'silent' \| 'error' \| 'warn' \| 'info' \| 'debug'` | `'silent'` | Console log verbosity |
| `logHandler` | `(level, message, meta) => void` | console | Custom log sink |
| `rateLimit` | `{ search?, browse?, stream?, autocomplete? }` (req/min) | sane defaults | Per-endpoint rate limit |
| `minRequestGap` | `number` (ms) | — | Minimum gap between requests |
| `cache` | `{ dir?, enabled?, ttl? }` | enabled, default TTLs | SQLite cache config |
| `cache.ttl` | `{ stream?, search?, home?, artist? }` (seconds) | see §15 | Per-cache TTL override |
| `visitorId` | `string` | auto-fetched | Pin a visitor ID (else rotated 30d) |
| `userAgent` | `string` | sane default | Override outbound UA |
| `language` | `string` | `'en'` | InnerTube `hl` |
| `location` | `string` | `'US'` | InnerTube `gl` |
| `proxy` | `string` (HTTP/HTTPS/SOCKS URL) | — | Network proxy |
| `maxRetries` | `number` | `3` | Retry attempts for transient errors |
| `backoffBase` | `number` (ms) | `1000` | Exponential backoff base |
| `backoffMax` | `number` (ms) | — | Cap on backoff delay |
| `sourceOrder` | `'best' \| SourceName[]` | `'best'` | Audio source preference |
| `cookiesPath` | `string` (path to Netscape cookies.txt) | — | Auth for age-gated/private |
| `youtubeApiKey` | `string` | — | Enables `YouTubeDataAPISource` fallback |
| `identify.acoustidApiKey` | `string` | — | Required for fingerprint identification |
| `identify.songrecBin` | `string` (path) | — | Optional Shazam fallback (tried first) |
| `poToken` | `string` | — | Static PoToken for InnerTube |
| `getPoToken` | `(videoId, client) => Promise<string \| null>` | — | Async PoToken generator (overrides `poToken` if both set) |
| `lyrics.providers` | `LyricsProviderSpec[]` | default chain | Override the 7-provider chain |

`LyricsProviderSpec = LyricsProviderName | { name, fetch }` — built-in name string OR custom provider.

## 3. Search

| API | Returns | Notes |
|---|---|---|
| `mk.search(query)` | `SearchResults` | Mixed songs/albums/artists/playlists |
| `mk.search(q, { filter: 'songs', limit?, source? })` | `Song[]` | |
| `mk.search(q, { filter: 'albums', limit?, source? })` | `Album[]` | |
| `mk.search(q, { filter: 'artists', limit?, source? })` | `Artist[]` | |
| `mk.search(q, { filter: 'playlists', limit?, source? })` | `Playlist[]` | |
| `mk.autocomplete(query)` | `string[]` | YouTube Music suggestions |

In-memory search cache: LRU 256 entries × 5-min TTL. `SEARCH_CACHE_MAX = 256` (private constant on `MusicKit`).

URL/query resolvers (call before passing to any method):

| Function | Returns |
|---|---|
| `resolveInput(input: string)` | Canonical ID or query string |
| `resolveSpotifyUrl(url: string)` | `Promise<string \| null>` — "Title Artist" search query, scraped from open.spotify.com |

`resolveInput` handles: `youtube.com/watch?v=ID`, `youtu.be/ID`, `music.youtube.com/watch?v=ID`, `music.youtube.com/browse/<browseId>`, `music.youtube.com/playlist?list=ID`, `music.youtube.com/search?q=QUERY`. Unrecognised input → returned unchanged.

## 5. Streaming

| API | Returns | Notes |
|---|---|---|
| `mk.getStream(videoId, { quality? })` | `StreamingData` | quality: `'high'` (default) or `'low'` |
| `mk.streamAudio(id)` | `Promise<NodeJS.ReadableStream>` | yt-dlp pipe — direct audio bytes |
| `mk.streamPCM(id)` | `Promise<NodeJS.ReadableStream>` | ffmpeg-decoded PCM (48kHz/16-bit/stereo) |

### Stream resolution chain (transparent fallback)

1. SQLite cache (~6h TTL, expiry-checked)
2. InnerTube multi-client: `ANDROID_VR → TVHTML5 → YTMUSIC` (`STREAM_CLIENT_FALLBACK_ORDER`)
3. yt-dlp shell-out (universal fallback)

Failed → next in chain. yt-dlp fallback emits `'retry'` event with `endpoint: 'stream'`.

### Stream-related helpers

| Function | Returns |
|---|---|
| `isStreamExpired(streamData)` | `boolean` — true if `expiresAt` ≤ now+5min |

### Stream URL fields

`StreamingData = { url, codec: 'opus' \| 'mp4a', mimeType, bitrate, expiresAt, loudnessDb?, sizeBytes? }`. Track type detection happens via `videoDetails.musicVideoType` (private uploads flagged but not surfaced in current `StreamingData`).

## 7. Lyrics — 7-Provider Chain

| API | Returns |
|---|---|
| `mk.getLyrics(id)` | `Lyrics \| null` — uses default chain, cached 10y |
| `mk.getLyrics(id, { providers })` | per-call override — bypasses cache |
| `mk.registerLyricsProvider(provider, position?)` | runtime registration |

`Lyrics = { plain: string, synced: LyricLine[] \| null, source?: LyricsProviderName }`
`LyricLine = { time, text, words?: WordTime[] }`
`WordTime = { time, duration?, text }` (alias: deprecated `LyricWord`)

### Built-in providers (default order)

| # | Name | Source | Strength |
|---|---|---|---|
| 1 | `better-lyrics` | lyrics-api.boidu.dev | Apple Music TTML, **only word-level provider** |
| 2 | `lrclib` | lrclib.net | Best line-level coverage globally |
| 3 | `simpmusic` | api-lyrics.simpmusic.org | Fan-aggregator, falls back to videoId |
| 4 | `youtube-native` | YT Music lyrics tab | Plain text, official labels |
| 5 | `kugou` | mobileservice.kugou.com + lyrics.kugou.com | Chinese music |
| 6 | `lyrics-ovh` | lyrics.ovh | Plain-text catch-all |
| 7 | `youtube-subtitle` | YT auto-captions | Universal last resort, filters non-lyric tags |

## 9. Audio Identification

| API | Returns |
|---|---|
| `mk.identify(filePath)` | `Promise<Song \| null>` |

Requires `config.identify.acoustidApiKey`. If `config.identify.songrecBin` is set, SongRec (Shazam) is tried **first**; falls back to Chromaprint+AcoustID on miss.

### Standalone `Identifier` class

```ts
const id = new Identifier({ acoustidApiKey, songrecBin?, fetch? })
const fp = await id.fingerprint(filePath)            // { fingerprint, duration }
const result = await id.lookup(fp.fingerprint, fp.duration)  // IdentifyResult | null
const songrecResult = await id.recognizeWithSongrec(filePath) // IdentifyResult | null
```

`IdentifyResult = { artist, title, score }`

## 13. Source Routing

The SDK abstracts streaming sources behind an `AudioSource` interface. Default source is `youtube` (InnerTube via `youtubei.js`).

| Knob | Effect |
|---|---|
| `config.sourceOrder = 'best'` | default — uses `['youtube']` |
| `config.sourceOrder = ['youtube', 'custom']` | explicit order |
| Per-call `{ source }` | `mk.search(q, { source: 'youtube' })`, `mk.getHome({ source })` |
| `config.youtubeApiKey` | enables `YouTubeDataAPISource` fallback |
| `mk.registerSource(audioSource)` | runtime custom source |

### `AudioSource` interface (for custom sources)

```ts
interface AudioSource {
  readonly name: string
  canHandle(query: string): boolean
  search(query, options?): Promise<SearchResults | Song[] | Album[] | ...>
  getStream(id, quality): Promise<StreamingData>
  getMetadata(id): Promise<Song>
  // optional
  getAlbum?, getArtist?, getPlaylist?
  getRadio?, getRelated?
  getHome?, getCharts?
  getMoodCategories?, getMoodPlaylists?
  autocomplete?
}
```

`SourceName` is currently `'youtube'` (custom names are user-defined strings).

## 14. Anti-Ban Infrastructure

| Feature | Config |
|---|---|

| PoToken support | static `config.poToken` OR async `config.getPoToken` |

|---|---|

## 15. Caching

SQLite-backed (`node:sqlite`, Node 22+ built-in — no `better-sqlite3` dep). Keyed by namespace.

| Cache type | Default TTL | Override |
|---|---|---|
| Stream URLs | 6h (`Cache.TTL.STREAM = 21600`) | `config.cache.ttl.stream` |
| Search results | 5min (`Cache.TTL.SEARCH = 300`) | `config.cache.ttl.search` |
| Home feed | 8h (`Cache.TTL.HOME = 28800`) | `config.cache.ttl.home` |
| Artist pages | 1h (`Cache.TTL.ARTIST = 3600`) | `config.cache.ttl.artist` |
| Lyrics | 10y (`Cache.TTL.LYRICS = 315360000`) | (fixed) |
| Visitor ID | 30d (`Cache.TTL.VISITOR_ID = 2592000`) | (fixed) |

| Knob | Effect |
|---|---|
| `config.cache.dir` | Custom cache directory |
| `config.cache.enabled = false` | Disable all caching |

### Standalone `Cache` class

```ts
const c = new Cache({ enabled: true, path?: string })
c.get<T>(key)                  // T | null
c.set(key, value, ttlSeconds)
c.delete(key)
c.isUrlExpired(url)            // boolean
c.sweep()                      // number — rows deleted
c.getStats()                   // { hits, misses, keys }
c.close()
```

## 16. Error Handling

| `NetworkError` | `statusCode?: number` | HTTP layer failures |
